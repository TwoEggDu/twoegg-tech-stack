这张模板是 [预算体检]({{< relref "performance/game-budget-01-budget-healthcheck.md" >}}) 的执行版。

它要解决的不是"项目做得完不完美"，而是更现实的问题：

`这项目现在到底有没有一套能运转的预算体系。`

## 体检表模板

| 检查项 | 结论 | 证据 | 风险等级 | 下一步动作 |
|-------|-----|-----|---------|-----------|
| 首包 / 全量包是否分开管理 |  |  |  |  |
| 稳态 / 峰值 / 红线是否分开管理 |  |  |  |  |
| CPU / GPU 帧时间是否有明确口径 |  |  |  |  |
| 场景 / 角色 / 特效是否有规则表 |  |  |  |  |
| 是否覆盖最低支持设备 |  |  |  |  |
| 是否有 baseline / CI / 回归动作 |  |  |  |  |
| 是否有超线动作和例外审批 |  |  |  |  |

## 结论分级建议

- `A`
  预算线完整，且已经接进执行。
- `B`
  有预算，但部分失真或缺少门禁。
- `C`
  有零散数字，没有成体系。
- `D`
  只有模糊愿望，没有预算体系。

## 怎么用这张表

1. 不要只问"有没有文档"。
2. 每项都要求看到证据：
   - 表
   - 构建产物
   - baseline
   - 验证记录
3. 给每项都写动作，不要只写"待优化"。

## 最短结论

预算体检模板的价值，不是形式化检查，而是：

`快速识别项目到底缺的是预算线、执行链，还是责任边界。`
