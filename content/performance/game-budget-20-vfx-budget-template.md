特效预算最容易写成“一个技能最多多少粒子”，但这很快就会失真。

更有用的写法，是让每个技能或一组战斗表现都有一张组合验收表。

## 模板字段

| 字段 | 数值 / 等级 | 备注 |
|------|------------|------|
| 发射器数量 |  |  |
| 最大活跃粒子数 |  |  |
| 最大屏占比 |  |  |
| 透明叠层风险 |  |  |
| 是否使用 RT / 扭曲 |  |  |
| 贴图 / 网格资源量 |  |  |
| 是否需要预热 |  |  |
| 对象池策略 |  |  |
| 出现时机 |  |  |
| 最低支持档位 |  |  |

## 为什么这些字段必须一起看

因为特效最容易同时打爆：

- CPU 模拟
- GPU overdraw
- 内存和对象池
- 关键时机的帧预算

## 怎么用

1. 先按技能或特效簇建表。
2. 再按目标设备档位写允许版本。
3. 验证时固定：
   - 单独释放
   - 战斗高峰叠加
   - 首次命中

## 最短结论

特效预算模板的价值，不是限制创作，而是让团队在做强反馈时知道：

`这次表现到底在花哪几笔账。`
