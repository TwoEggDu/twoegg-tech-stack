分档配置做完之后，很多团队会做一件事：拿低档设备跑一遍游戏，然后说"能跑，没问题"。

但有一个更重要的问题没有被问到：**低档的玩家感觉这还是同一款游戏吗？**

这不是在说低档要和高档一样漂亮——那是不可能的，也没必要。问题是：当你把游戏所有的视觉参数降低之后，它的核心感觉还在吗？色调还是那个色调，角色还是那个角色，打击感还是那个打击感？

很多游戏低档版本质量下降的真正问题不是"画质数字低了"，而是"这个游戏变味了"。

---

## 视觉质量 vs. 视觉身份

先把两个概念分开。

**视觉质量**（Visual Quality）：可以量化的画质指标。Shadow Map 分辨率、粒子数量、Render Scale、贴图尺寸。这些东西降低是分档的必要代价，玩家会感知到，但通常理解并接受——因为他们知道自己用的是低端手机。

**视觉身份**（Visual Identity）：让这款游戏看起来"是这款游戏"的核心元素。色调风格、角色辨识度、标志性的特效风格、UI 的质感。这些东西一旦被改变，不是"画质变差了"，而是"这个游戏变成另一个游戏了"。

玩家对视觉质量下降的容忍度远高于你的预期。但对视觉身份的破坏，他们会立刻感知到，即使他们说不清楚哪里不对。

---

## 找到你游戏的视觉身份元素

不同游戏的视觉身份来自不同的地方。在做低档配置之前，应该先问：

**这款游戏靠什么让玩家一眼认出它来？**

几个常见的来源：

### 色调与氛围

如果你的游戏有强烈的色调特征——比如高饱和度的日系 ACG 风格、暗沉的欧美奇幻风格、明亮的卡通色彩——那么 Color Grading 是你的视觉身份的一部分。

操作：低档用预烘焙 LUT 保留色调，不要关掉 Color Grading。一张预烘焙 LUT 的运行时代价接近零，没有任何理由在低档关掉它。

### 角色辨识度

如果你的游戏核心是角色（卡牌、ARPG、养成类），角色的清晰度和细节是玩家最关注的部分。角色的主贴图、面部细节、标志性的轮廓形状，是视觉身份的直接载体。

操作：角色主贴图的分辨率在低档也要保住（1024 ASTC 6x6），不要砍到 512。场景贴图可以激进压缩，但角色不行。

### 技能特效风格

如果你的游戏有标志性的技能视觉风格（比如某个主角的技能一定是蓝色流光、某个职业的技能一定有特定的粒子形状），这是游戏世界观的视觉表达，也是玩家情感连接的来源。

操作：低档砍粒子密度时，要保住核心技能的"形状"和"颜色"，而不是只砍数量。一个密度只有 20% 但形状和颜色正确的技能，比一个密度 80% 但色彩和形状都变了的技能，感知上更像"同一个技能"。

### 打击感与反馈

打击感不只是音效，屏幕的抖动、命中时的粒子爆发、受击者的位移，都是视觉身份的组成部分。在动作游戏里，打击感是"游戏质量"最直接的感知来源。

操作：屏幕抖动（Camera Shake）和命中粒子是低代价的效果，不要在低档关掉。关掉的感知代价远大于它省下的 GPU 预算。

## 身份不只是视觉：听觉和动画也在传达"是不是同一款游戏"

低档配置在讨论时容易被收成"画质讨论"，但玩家感知"同一款游戏"时，画面只是其中一层。**音效和动画的降级同样能把游戏感觉做没**。

### 听觉身份

低档配置对音频的常见处理是降采样率、砍音轨层数、关空间音效。这些操作大部分情况下省的 CPU / 内存极其有限，但感知代价很大。

| 音频元素 | 能动吗 | 说明 |
|---------|--------|------|
| 标志性音效（技能、命中、击杀、胜利）| 否 | 玩家对这几个音效的记忆强度 ≈ 对主角贴图的记忆强度 |
| BGM 完整度 | 否 | 一旦砍掉分层（比如低档只保留主旋律 stem），氛围立刻变薄 |
| 环境音 layer 数量 | 可以砍 | 从 5 层背景音砍到 2 层，多数玩家感知不到 |
| 3D 空间音效 | 可以关 | 只在戴耳机玩的玩家感知强，低档普遍外放，感知弱 |
| 音频采样率 | 极少数情况下可降 | 44.1kHz → 22kHz 对人声和主音效影响明显，慎用 |

操作要点：**砍音频层数而不是砍音频质量**。层数是环境音饱满度，多数玩家不敏感；质量是主旋律和关键音效的清晰度，所有玩家都敏感。

### 动画身份

动画承载的是游戏的节奏感。低档配置最容易踩的陷阱是降动画采样率或砍骨骼数量——前者影响"动作流畅度"，后者影响"动作准确度"。

| 动画元素 | 能动吗 | 说明 |
|---------|--------|------|
| 主角战斗动画帧率 | 否 | 主角动画 30fps → 15fps，玩家会直接感觉"卡"而不是"低画质" |
| 主角骨骼 LOD | 否 | 角色骨骼砍半会让关键动作（如出招起手）丢失辨识度 |
| NPC 动画帧率 | 可以砍 | 远处 NPC 15fps 完全够用 |
| NPC 骨骼 LOD | 可以砍 | 远处 NPC 骨骼数减半不影响辨识 |
| 镜头运镜节奏 | 否 | 技能镜头、死亡慢镜、开场镜头的节奏是叙事语言的一部分 |
| 布料 / 头发解算 | 可以关 | 低档用预烘焙 morph 或静态衣摆，代价低感知低 |

核心原则：**主角的一切动画都要保留节奏和辨识度，远处和次级对象可以任意降**。节奏感（动画帧率）比精度（骨骼数）更优先保留——玩家宁可接受"动画有一点刚性"，也不能接受"动作看起来慢半拍"。

## 按游戏类型的视觉身份清单

不同类型游戏的"不能动"列表不一样：

### 动作 / ARPG

| 视觉元素 | 能动吗 | 说明 |
|---------|--------|------|
| 角色主贴图分辨率 | 否 | 低档保 1024 |
| 核心技能特效形状和颜色 | 否 | 可以降粒子数，不能改形状 |
| 打击反馈（Camera Shake / 命中粒子）| 否 | 代价低，感知高 |
| 实时阴影 | 可以换方案 | 改 Blob Shadow，不能全关 |
| 场景背景贴图 | 可以砍 | 玩家关注点在角色，不在背景 |
| SSAO | 可以关 | 动作游戏中 AO 感知弱 |

### 开放世界 / MMO

| 视觉元素 | 能动吗 | 说明 |
|---------|--------|------|
| 整体光照色调（黄昏/黎明等）| 否 | 氛围来源，LUT 必须保留 |
| 近处地形和建筑细节 | 部分可动 | 30m 以内保住，30m 以外可压 |
| 角色外观（装备高光、皮肤细节）| 否 | MMO 玩家极度关注角色外观 |
| 植被密度 | 可以砍 | 低档 40%，极低档可以关 |
| 远处场景质量 | 大幅砍 | 玩家通常看不到细节 |
| NPC 同屏数量 | 可以砍 | 低档 8-10 个，不影响核心体验 |

### 卡牌 / 回合制 RPG

| 视觉元素 | 能动吗 | 说明 |
|---------|--------|------|
| 角色立绘质量 | 否 | 这是核心商业资产，全档保 2048 |
| 卡牌动效 | 部分可动 | 主要角色的动效不能删，NPC 可以简化 |
| 战斗特效的颜色和风格 | 否 | 可以用 Flipbook 替代粒子，但风格不能变 |
| 背景场景 | 大幅砍 | 玩家注意力在角色和手牌，不在背景 |
| UI 动画 | 可以简化 | 但不能删，UI 动画是品质感的重要来源 |

### 休闲 / 超休闲

| 视觉元素 | 能动吗 | 说明 |
|---------|--------|------|
| 整体色彩饱和度和活泼感 | 否 | 色调是休闲游戏的情绪来源 |
| 核心 Gameplay 反馈特效 | 否 | 消消乐的消除特效、跑酷的速度拖影，是玩法的一部分 |
| 背景细节 | 大幅砍 | 休闲游戏焦点极度集中，背景几乎不被关注 |
| 角色精细度 | 可以砍 | 休闲游戏角色通常是简化风格，512 贴图够用 |

## 一个典型反面案例：通用规则砍出来的"同名不同游戏"

下面是一段虚构但贴近真实工作流的对比，说明同一款 ARPG 做低档配置时，按通用规则砍 vs. 按身份清单砍，结果能差出多远。

**背景**：一款主打"红蓝流光剑气"的 ARPG，目标海外新兴市场，底线机型 Redmi 9A 级别（Adreno 505）。

### 方案 A：按通用规则砍

团队照搬了一份常见的"低档配置模板"：

- Color Grading 关闭（"LUT 计算贵"）
- 所有实时阴影关闭（"Shadow Map 代价高"）
- 粒子密度统一砍到 30%
- 贴图全部降到 512 ASTC 8x8
- 关闭 Bloom 和 Camera Shake

跑起来能稳 30fps，包体体积也下来了。但玩家社区的反馈是：

- "低配版的剑气没有流光了，感觉就是个普通刀"
- "我朋友发我的截图和我自己游戏里看到的完全不是一个画面"
- "打击感很弱，像点图标不像打架"
- "这个游戏原本那种偏蓝紫的氛围不见了，看起来灰灰的"

这些反馈都指向同一件事：**砍的是身份，不是质量**。

### 方案 B：按身份清单砍

重做后的配置：

| 配置项 | 方案 A（错）| 方案 B（对）| 原因 |
|-------|-----------|-----------|------|
| Color Grading | 关闭 | 预烘焙 LUT 保留 | 色调是身份 |
| 主角实时阴影 | 关闭 | 保留 512 Shadow Map，Distance 缩到 15m | 主角不能悬浮 |
| 远处实时阴影 | 关闭 | 关闭，改为烘焙 | 远处玩家不关注 |
| 剑气特效粒子密度 | 砍到 30% | 砍到 40%，但形状和颜色（红蓝流光）完全保留 | 标志性特效是身份 |
| 背景装饰粒子 | 砍到 30% | 砍到 0%（全关）| 装饰不是身份，可以激进 |
| 主角贴图 | 512 | 1024 ASTC 6x6 | 主角辨识度 |
| 远景贴图 | 512 | 256 ASTC 8x8 | 远景不影响身份 |
| Bloom | 关闭 | 1/4 分辨率保留 | 流光特效依赖 Bloom 才能"亮起来" |
| Camera Shake | 关闭 | 保留（代价几乎为零）| 打击感 |

方案 B 的 GPU 预算比方案 A 多了大约一点点（主要来自 Shadow Map 和 Bloom），但在 Adreno 505 级别设备上依然能稳定 30fps——代价在"可以接受"范围。

**收益**：玩家论坛的"不是同一款游戏"类投诉几乎消失。低档玩家仍然知道自己在玩的是那个"红蓝流光剑气"的游戏。

这个对比想说明的是：**低档配置的质量感，不来自"画质参数降得少"，而来自"身份元素没被错杀"**。按通用模板砍很方便，但通用模板不知道你的游戏是谁——这件事只有做游戏的人自己能说清楚。

## 一个检验方法：截图盲测

在做完低档配置之后，可以用这个方法验证视觉身份是否被保住：

1. 截一张高档的游戏截图和一张低档的游戏截图
2. 把这两张图发给没有参与开发的人（内部其他部门，或者玩家社区小范围测试）
3. 问："这两张图，你觉得是同一款游戏吗？"

如果答案是"看起来不太一样，像两款不同的游戏"，说明低档砍到了视觉身份上；如果答案是"是同一款，但画质有差别"，说明砍的方向是对的。

这个测试代价极低，但能发现很多配置问题——尤其是 Color Grading 被关掉、角色贴图降幅过大、核心特效被错误地关闭这类问题。

## 分档配置里最常见的"变味"操作

| 错误操作 | 为什么会变味 | 正确做法 |
|---------|------------|---------|
| 低档关掉 Color Grading | 色调消失，游戏变"灰"，风格感消失 | 换预烘焙 LUT，不关 |
| 角色主贴图降到 512 | 角色变模糊，辨识度降低，"主角变路人" | 保 1024，让场景背景去承担压缩代价 |
| 核心技能特效全部关掉 | 技能变成纯数字，游戏失去冲击感 | 换低代价 Flipbook，保住形状和颜色 |
| Camera Shake 关掉 | 打击感消失，战斗变成"点图标" | 代价极低，全档保留 |
| 近处 LOD 切换过激 | 角色/物体在玩家眼前突然"变形" | 只对远处激进，近处 LOD 1–2 层保住 |
| 光照色温改变 | 白天/黑夜的氛围感消失 | 低档可以降精度，不能改色温本身 |

## 开发期就能避免：让设计天生带着低档版本

视觉身份被错杀，往往不是决策失误，而是**工作流没有让低档机出现在设计决策的现场**。绝大多数美术、TA、策划的评审都发生在开发机（旗舰或准旗舰）上，低档配置是在版本接近发布时由平台组单独做——这时已经很难改身份级的决策。

更稳的做法是把低档放进日常工作流：

### 1. 每个效果在提需求阶段就定义三档表现

TA 和美术合作定效果时，除了高档目标，应该同时回答：

- 中档削掉什么（通常是质量参数）
- 低档削掉什么（通常是数量或后处理）
- 极低档彻底换成什么替代方案

如果回答不了"低档的这个效果长什么样"，这个效果还不该进项目。

### 2. 美术 review 有一轮在低档机上过

新角色、新场景、新特效的评审阶段，准备一台真实低档机（Adreno 610 级别足够），把同一组资产在低档配置下跑一遍。这一步不是为了找性能问题，而是为了验证"低档版看起来还像这个游戏吗"。

这个 ceremony 每周一次就够，代价很小，但能在问题还在工作室阶段就暴露出来，而不是等到发布后玩家论坛反馈。

### 3. 截图自动化：每次提交构建都存一组低档截图

CI 里加一步自动在低档配置下跑黄金场景并截图，归档到构建报告里。美术和 TA 每周扫一眼，就能看到"低档版本"在版本间的演化轨迹。这比"等低档配置做完再集中看"早半年发现问题。

### 4. 标志性效果单独挂 owner

色调、主角、标志性技能特效、打击感——这四类身份元素应该有明确的 owner（不是每个都同一人）。任何人想改这四类的低档配置，必须过 owner 这一关。

这是反"无声腐化"的关键——没 owner 的身份元素会在版本迭代中被静默砍掉，等发现时已经过了能简单回滚的时机。

---

## 结论

分档的目标不是让低档设备玩到一款"降级版游戏"，而是让他们玩到**同一款游戏的轻量版本**。

区别在于：前者是不同的产品，后者是同一个产品在不同硬件上的表现。

要做到这一点，需要在每次降质决策前问：**这个东西是画质数字，还是这款游戏是谁？** 前者可以砍，后者不行。
