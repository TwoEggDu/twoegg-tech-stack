角色预算最容易被简化成：

- 主角多少面
- NPC 多少面

但真正能执行的角色预算表，必须把单体规格和同屏规模写在一起。

## 模板字段

| 字段 | 主角 | Boss | 精英 | 普通 NPC |
|------|------|------|------|---------|
| 面数 / 顶点数 |  |  |  |  |
| 材质球数量 |  |  |  |  |
| 骨骼数 |  |  |  |  |
| 蒙皮影响数 |  |  |  |  |
| 贴图套数 / 分辨率 |  |  |  |  |
| Shader 特性 |  |  |  |  |
| LOD 层级 |  |  |  |  |
| 同屏数量 |  |  |  |  |

## 为什么这张表比面数表更有用

因为角色真实成本来自：

- 几何
- 材质
- 动画和蒙皮
- 贴图
- Shader
- 同屏叠加

## 怎么用

1. 先按角色类型填单体规格。
2. 再按玩法填同屏数量。
3. 最后按设备档位给出可接受版本。

## 最短结论

角色预算模板真正值钱的地方，是把：

`单体规格 x 同屏数量 x 目标档位`

写成一张大家都能拿来验收的表。
