> 如果只用一句话概括这篇文章，我会这样说：UI 很少是“平时一直很重”，更常是首开、切页、叠层和过渡时，把一串本来分散的成本同时拉了起来。

这篇面向 UI 美术。  
它不先讲 Canvas、Rebuild 或 Atlas 的底层实现，而是先讲 UI 最容易在哪些时刻和哪些表达方式上出问题。

这篇要回答的是：

`为什么 UI 看起来不像 3D 场景那么重，却总在首开、切页和叠层时把体验打坏？`

## UI 的风险通常不在“常驻”，而在“切换”

很多团队会把 UI 问题理解成：

`这个界面节点太多。`

这句话并不总是错，但它不够完整。  
UI 真正最危险的时刻，通常是：

- 第一次打开
- 页面切换
- 弹窗叠层
- 同时拉启动效和数据填充

也就是说，UI 很多时候像一条小型加载链，而不是一个静态页面。

所以 UI 美术第一层该看的，不是“这页静态截图漂不漂亮”，而是：

- 它怎么打开
- 怎么切换
- 会不会叠层
- 一次要拉起多少东西

## 图集、模糊背景和遮罩为什么最容易出问题

### 图集问题很少在单张图上暴露

UI 美术最容易遇到的情况是：

- 单张图看起来都合理
- 但页面一开，资源准备和切换就开始出问题

这类问题往往不是在提醒你“某一张图太大”，而是在提醒你：

`这一页的图像组织方式，对首开和切页不友好。`

### 模糊背景和大面积半透明是高风险项

模糊背景、柔边遮罩、半透明大底图、全屏叠色层，特别容易被当成 UI 气质的默认组成。

但它们也是最值得多问一句的东西：

- 是不是一定要全屏
- 是不是一定要常驻
- 是不是所有弹窗都默认带

如果答案是“都带”，那风险通常不小。

### 遮罩不是不能用，而是不能一路叠

单个遮罩不一定可怕。  
危险的是层层嵌套、弹窗再套弹窗、列表里再带复杂动态元素。

UI 美术不需要先懂底层实现，也能先建立一个很实用的判断：

`只要层级、遮罩、透明和动效一起上，首开和切页就要格外小心。`

## 动效和层级为什么总在切页时集中出事

UI 动效往往不是问题本身。  
问题在于它太喜欢和别的事情一起发生。

最常见的现场是：

- 页面第一次打开
- 节点批量激活
- 动效同时开跑
- 数据和文本一起刷新
- 弹窗背景和遮罩一起叠上来

这时候玩家看到的是：

`怎么这一下突然抖了。`

所以 UI 动效最危险的不是“有没有动”，而是：

- 是不是所有东西都同时动
- 是不是第一次打开就把所有内容一起亮出来
- 是不是一边切页一边还在叠层

## 哪些 UI 表达最容易被误判成“只是审美选择”

### 误判一：大面积模糊背景只是风格，不算性能问题

不稳。  
它很容易成为高风险项，尤其当它被频繁叠层和切换时。

### 误判二：首开抖一下没关系，反正第二次就好了

这不稳。  
首开正是玩家最容易记住问题的时刻。

### 误判三：UI 不像 3D 场景那么复杂，所以应该没那么危险

也不稳。  
UI 的伤害往往更直接，因为它正好发生在用户交互最密的时候。

### 误判四：动效越完整，品质感越好

品质感当然重要，但真正该保的是节奏、层次和核心反馈，不是所有节点都必须同时带完整过渡。

## UI 提交前自检表

| 自检项 | 看到什么信号要警惕 | 建议动作 |
|------|----------------|---------|
| 1. 这个页面是不是首开最危险 | 第一次打开明显抖，第二次好很多 | 优先把它当首开时机问题看 |
| 2. 图像组织是不是对首开不友好 | 单图不大，但页面一开就明显吃力 | 先确认图集和页面资源组织方式 |
| 3. 模糊背景和大面积透明是不是默认常驻 | 每个弹窗都带柔边、模糊、叠色大底 | 先确认哪些是核心风格，哪些能降 |
| 4. 遮罩和层级是不是一路往上叠 | 弹窗套弹窗，列表里再带复杂动态项 | 提前警惕叠层风险，不要只看单页 |
| 5. 动效是不是都在同一时刻启动 | 首开、切页时所有内容一起动 | 优先考虑分阶段展示 |
| 6. 切页是不是比常驻更危险 | 常驻没事，切页或回页时明显抖 | 重点检查切换而不是静态页面 |
| 7. 这个页面是不是用户高频入口 | 背包、任务、商店、活动页反复打开 | 高频入口比低频入口更要保守 |
| 8. 这已经是 UI 资源问题还是展示规则问题 | 涉及首开顺序、层级规则、分阶段展示 | 拉 UI 程序、TA 一起定边界 |

## 结论

UI 最容易出问题的地方，不是“它像 3D 一样很重”，而是：

- 它特别容易在首开和切页时把很多成本一起拉起来。
- 它特别容易把层级、遮罩、模糊和动效叠在同一时刻。
- 它正好又发生在玩家交互最密的时刻。

如果你先用这个视角看 UI，很多“为什么这页一打开就抖”的问题，会比只盯节点数量更容易看清。
