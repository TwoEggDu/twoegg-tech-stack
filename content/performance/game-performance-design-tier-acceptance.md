> 如果只用一句话概括这篇文章，我会这样说：分档验收看的不该只是"能不能跑"，而是低中高档玩家玩到的，是不是仍然是同一款游戏。

这篇文章解决的是策划最容易模糊的一件事：

`低中高档到底该保什么，不该保什么，验收时又该怎么判断"只是降级"还是已经"变味"。`

很多团队谈分档时，会天然把话题收缩成：

- 帧率够不够
- 分辨率降了多少
- 某个效果开没开

这些当然重要，但它们还不够。

对策划来说，更关键的是：

- 玩家是不是还认得这套玩法和反馈。
- 关键目标和信息是不是还清楚。
- 核心战斗节奏和情绪是不是还成立。
- 低档是不是只是"轻量版"，而不是"另一款游戏"。

## 验收分档一致性，看的不是"能不能跑"，而是"还是不是同一款游戏"

"能跑"只是底线，不是验收结论。

因为很多分档问题都能在技术层面勉强过关，但玩家一上手就会觉得不对。

常见例子包括：

- 低档技能还能放，但关键识别感没了。
- 主城还能进，但热闹感和空间层次被抽空了。
- UI 还能开，但层级混乱、反馈变钝、节奏变拖。
- 战斗还能打，但命中、危险提示、奖励反馈都明显失真。

这些问题不一定让系统报错，却会让体验"变味"。

所以分档验收的第一原则不是：

`这个档位能跑到多少帧？`

而是：

`这个档位还保住了哪些不可动的体验边界？`

## 三个档位各自该保什么

为了避免每次验收都靠感觉，我建议先把高、中、低档的职责定清楚。

| 档位 | 必须保住的东西 | 可以接受的降级 | 不该发生的事 |
|------|--------------|--------------|-------------|
| 高档 | 完整风格、完整反馈、完整密度、完整演出层级 | 少量极端场景下的局部取舍 | 核心表现缺失、主要效果被明显裁掉 |
| 中档 | 视觉身份、核心反馈、主要识别点、主要氛围 | 背景密度下降、装饰层简化、次级动效减少 | 核心技能认不出来、主路径 UI 变慢、主要情绪断层 |
| 低档 | 可玩性、可读性、关键反馈、主循环节奏、主要风格身份 | 大量装饰层下调、远景和背景复杂度下调、次级特效和 crowd 下调 | 玩起来像另一套规则、重要信息看不清、关键反馈缺席、低档必须靠猜 |

这张表最关键的一点是：

`低档不是"什么都能砍"，而是"该砍装饰层，不该砍身份层和可玩层"。`

## 哪些东西能降，哪些东西不能变味

策划做分档验收时，最容易卡住的地方就是：

`这个到底算核心，还是只算装饰？`

我建议先用下面这张表对齐口径。

| 必须保住 | 可以优先降级 |
|------|------------|
| 核心技能和危险提示的识别性 | 背景装饰密度 |
| 命中、受击、奖励等关键反馈 | 次级粒子和装饰型特效 |
| 主流程 UI 的可读性和响应顺序 | 非核心页面的附加动效 |
| 角色和阵营的基本辨识度 | 远景细节和 crowd 数量 |
| 游戏整体的主色调、氛围身份 | 阴影精度、装饰后处理、次级氛围层 |
| 玩家做决策所依赖的重要信息 | 不影响判断的精美度细节 |

这里真正要警惕的，不是"低档画面没高档漂亮"，而是：

- 低档玩家看不清该躲什么。
- 低档玩家认不出技能和角色。
- 低档玩家打不开高频页面或打开后节奏明显变慢。
- 低档玩家玩到的战斗节奏和高档已经不是同一种体验。

## 策划最常见的四种验收误判

### 误判一：帧率过线了，就算验收通过

不够。  
帧率只回答"跑得动"，不回答"玩起来是不是同一款游戏"。

### 误判二：看静态截图差不多，就说明一致

不稳。  
很多变味发生在动态反馈、切页节奏、技能识别和战斗手感里，截图看不出来。

### 误判三：只在空场景或轻场景下验收

也不稳。  
玩家最容易感知差异的，往往正是主城高密度、战斗高峰、复杂 UI 首开这些重场景。

### 误判四：低档本来就差一点，差很多也能接受

这句话最危险。  
低档允许"轻量"，不允许"换味"。如果差到要重新学习规则、重新猜信息，那就不是正常降级。

## 一套可以真正执行的分档验收流程

分档验收最好别停留在"大家觉得差不多"。  
更稳的做法，是固定几个关键场景，用同一套观察口径比较高、中、低档。

### 第一步：先固定 4 类关键场景

我建议至少固定这四类：

| 场景 | 为什么必须测 |
|------|-------------|
| 主城 / 大厅 / 活动入口 | 最容易暴露密度、导航和热闹感差异 |
| 高峰战斗 | 最容易暴露识别、反馈和节奏差异 |
| 高价值 UI | 最容易暴露首开、切页和层级差异 |
| 首次进入 / 首次触发 | 最容易暴露时机问题和第一次成本 |

### 第二步：高、中、低档跑同一段流程

不要每档随便看两眼。  
要让它们走同一段体验：

- 同一条主路径
- 同一场战斗
- 同一套技能
- 同一个页面入口
- 同一个奖励收取和退出过程

只有这样，差异才是可比较的。

### 第三步：不只看"差多少"，更看"差在哪一层"

验收时建议至少分三层记录：

- 身份层：这个档位看起来还是不是这款游戏。
- 可玩层：该看的信息、该做的判断是不是还成立。
- 装饰层：哪些只是精美度下降，但不影响核心体验。

只要问题落在前两层，就不该被算成"正常降级"。

### 第四步：把结论落成可复用口径

每次验收后，最好别只留一句"通过 / 不通过"。

更稳的写法是：

- 这个档位必须保住哪些体验。
- 哪些东西已经证明可以安全降级。
- 哪些东西一降就会变味，后续版本不要再反复试错。

这样后面做新活动、新角色、新页面时，团队才不会每次都从头吵一遍。

## 上线前至少过这张分档验收清单

| 检查项 | 验收时先问什么 |
|------|---------------|
| 核心战斗识别 | 低档还能一眼看懂危险、技能和命中结果吗？ |
| 主路径 UI | 高频页面打开、切页、确认的节奏还顺吗？ |
| 角色 / 阵营辨识 | 低档会不会把关键角色和阵营信息洗平？ |
| 主城 / 活动热闹感 | 降级后是变轻了，还是直接空了、散了？ |
| 首次体验 | 首进、首开、首战有没有因为降级变得更割裂？ |
| 奖励与成就反馈 | 低档拿到关键奖励时，情绪有没有被削没？ |
| 低档规则是否变了 | 玩家是不是必须改用另一套理解方式才能玩？ |
| 差异是否可复述 | 团队能不能清楚说出"这是正常降级，不是变味"的理由？ |

## 结论

策划验收分档时，最重要的不是把每个开关都背下来，而是先守住一条线：

`低中高档可以轻重不同，但不能让玩家玩到不同的产品。`

只要这条线清楚了，很多判断都会容易很多：

- 该保的是身份层和可玩层。
- 该降的是装饰层和密度层。
- 该测的是关键场景，不是空场景。
- 该记录的是可复用边界，不是一次性的感觉。

如果你先把这套口径立起来，后面的分档讨论就不会再反复掉回"低档本来就差一点"这种模糊争论里。
