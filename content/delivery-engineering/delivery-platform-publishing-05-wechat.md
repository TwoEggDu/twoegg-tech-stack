## 微信小游戏发布流程

```
1. miniprogram-ci upload（CI 自动上传）
2. 微信开发者工具 / 管理后台提交审核
3. 审核（1-3 天）
4. 审核通过 → 手动发布 或 定时发布
```

### 与原生 App 发布的核心差异

| 维度 | 原生 App | 微信小游戏 |
|------|---------|-----------|
| 分发渠道 | 应用商店 | 微信内嵌 |
| 安装方式 | 用户主动下载安装 | 打开即用，无安装步骤 |
| 更新方式 | 用户到商店更新 | 微信自动更新（静默） |
| 版本回撤 | 停止新版本分发 | 可以回退到任意历史版本 |
| 灰度能力 | iOS 无原生灰度 / Android 有 Staged Rollout | 微信后台原生支持灰度 |

**微信的版本回撤比原生平台灵活得多**——可以在后台直接回退到任意历史版本，且已有用户下次打开时自动获取回退后的版本。这是微信小游戏在交付安全性上的优势。

### 微信审核要点

| 要点 | 说明 |
|------|------|
| **内容合规** | 游戏内容符合微信平台规范（无暴力/赌博/政治敏感） |
| **用户体验** | 不能强制关注公众号、不能强制分享才能继续玩 |
| **隐私协议** | 必须有隐私协议弹窗，用户同意后才能收集数据 |
| **登录方式** | 必须支持微信登录，不能只有手机号登录 |
| **支付** | 必须使用微信支付，不能引导到第三方支付 |
| **包体限制** | 主包不超过限制（4MB/8MB/12MB） |

### 微信灰度发布

微信后台原生支持灰度发布，比 iOS 和 Android 都方便：

- 按比例灰度（1% → 10% → 50% → 100%）
- 按地区灰度（先在某些城市发布）
- 按用户 ID 灰度（指定测试用户优先体验）

灰度期间的版本切换是即时的——不需要等待审核。

### 微信特有的运营约束

微信小游戏的运营能力受微信平台规则约束：

- **分享限制**：诱导分享（"分享后获得奖励"）可能被平台处罚
- **广告限制**：广告展示频率和位置有平台规范
- **推送限制**：不能无限制推送消息，需要用户授权订阅消息
- **支付分成**：微信抽取 40% 的 iOS 内购分成（Android 和 iOS 走微信支付时分成比例不同）

这些运营约束不直接影响技术交付，但影响产品策略和发布计划。

## 小结

- [ ] CI 是否集成了 miniprogram-ci 自动上传
- [ ] 审核驳回原因是否有知识库记录
- [ ] 灰度发布是否利用了微信后台的原生灰度能力
- [ ] 版本回撤流程是否团队知晓
- [ ] 运营约束是否在产品设计阶段就纳入考虑
