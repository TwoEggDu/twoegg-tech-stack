## 这篇解决什么问题

V01-V09 覆盖的交付链路以客户端为主。从 V10 开始进入服务端交付——一条平行但本质不同的链路。这一篇先把两者的差异讲清楚。

## 五个本质差异

### 差异一：构建产物的形态

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 产物 | .ipa / .apk / WebGL 目录 | Docker 镜像 / 可执行文件 |
| 目标环境 | 用户设备（数百种型号） | 服务器集群（统一环境） |
| 平台差异 | iOS / Android / 微信（差异巨大） | Linux x64（基本统一） |

服务端不需要处理多端差异——只需要构建一份产物。但服务端的部署环境比客户端复杂——涉及负载均衡、数据库、缓存、消息队列等基础设施。

### 差异二：更新机制

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 更新决定权 | 用户选择是否更新 | 运维决定何时更新 |
| 更新方式 | 用户下载新版本 | 滚动更新 / 蓝绿部署 |
| 版本共存 | 可能有多个版本同时在线 | 通常只有一个版本在线（更新过程中短暂两版本共存） |
| 回滚 | 不能强制已更新用户回退 | 可以分钟级回滚到上一版本 |

服务端更新的最大优势是**可控性**——运维可以选择在低峰期更新，可以分钟级回滚。客户端一旦推送给用户，就无法收回。

### 差异三：状态管理

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 状态位置 | 用户设备本地 | 服务器内存 + 数据库 |
| 更新时的状态 | 重启后重新加载 | 必须保持在线用户的状态 |
| 数据迁移 | 本地存档格式兼容 | 数据库 Schema 迁移 |

服务端更新最难的部分不是"部署新代码"，而是"在不丢失在线用户状态的情况下更新代码"。这个问题在客户端不存在——客户端更新时用户会重启应用。

### 差异四：扩展模型

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 扩展 | 不需要（每个用户一个实例） | 水平扩展（加更多服务器节点） |
| 负载均衡 | 不需要 | 必须 |
| 有状态扩展 | 不适用 | 极其困难（游戏服务器通常有状态） |

游戏服务端通常是有状态的——每个游戏世界 / 房间 / 战斗 都有内存中的实时状态。有状态服务的扩展比无状态服务困难得多，因为状态不能简单地在节点间迁移。

### 差异五：监控维度

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 核心指标 | Crash 率、帧率、内存 | QPS、延迟、错误率、CPU/内存使用率 |
| 可观测性 | 依赖 SDK 上报 | 可以直接访问日志和指标 |
| 问题定位 | 需要用户环境信息 | 可以直接查看服务器日志 |

服务端的可观测性远好于客户端——可以直接查看日志、指标、追踪链路。但服务端问题的影响面更大——一个服务端 Bug 可能影响所有在线用户，而客户端 Bug 通常只影响特定条件的设备。

## 两条链路的协调点

虽然客户端和服务端是两条独立的交付链路，但它们在以下环节必须协调：

| 协调点 | 问题 | 处理方式 |
|--------|------|---------|
| **协议版本** | 客户端和服务端的通信协议必须兼容 | 协议版本号 + 向后兼容设计 |
| **发布时序** | 服务端先更新还是客户端先更新 | 通常服务端先更新（同时兼容新旧客户端） |
| **配置同步** | 游戏配置在客户端和服务端都需要 | 统一的配置版本号 + 同步机制 |
| **数据格式** | 存档、排行榜等数据格式变更 | 数据格式版本号 + Schema 迁移 |

**服务端先更新的原则**：服务端更新后必须同时支持旧版客户端和新版客户端。等客户端全量更新后，服务端才能停止对旧协议的支持。

## 服务端交付飞轮

服务端的交付飞轮和客户端共享相同的八个环节（生产→集成→构建→验证→发布→监控→闭环→改进），但每个环节的具体工程实践不同：

| 环节 | 客户端 | 服务端 |
|------|--------|--------|
| 构建 | 多端构建（iOS/Android/WebGL） | 单一构建（Docker 镜像） |
| 验证 | 设备矩阵测试 | 负载测试 + 集成测试 |
| 发布 | 平台审核 + 灰度 | 滚动更新 / 蓝绿部署 |
| 监控 | Crash 率 + 帧率 | QPS + 延迟 + 错误率 |
| 回滚 | 无法强制回退 | 分钟级回滚 |

V10-V12 会逐一展开服务端在每个环节的工程实践。

## 小结

- [ ] 是否理解客户端和服务端交付的五个本质差异
- [ ] 协议版本兼容是否有设计（服务端同时支持新旧客户端）
- [ ] 发布时序是否有协调策略（通常服务端先更新）
- [ ] 配置变更是否有客户端和服务端的同步机制
