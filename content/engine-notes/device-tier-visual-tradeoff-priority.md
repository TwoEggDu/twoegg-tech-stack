分档配置里最难的决策不是"高档开什么"，而是**当性能预算不够用时，到底关哪个**。

很多团队在做低档配置时的做法是：哪个效果看起来"贵"就关哪个，或者跟着别的游戏抄配置。这两种方法的问题是一样的——它们都没有真正回答这个问题：**这个效果关掉之后，玩家会不会觉得游戏变差了？**

GPU 代价高的效果不一定会被玩家注意到。玩家感知强的效果不一定 GPU 代价高。把这两个维度放在一起看，才能找到真正的性价比最优解。

---

## 决策矩阵

先建一个框架，把常见视觉效果按两个维度排：

- **横轴**：GPU 代价（帧预算占比）
- **纵轴**：玩家感知敏感度（关掉之后会不会被注意到）

```
玩家感知
  高  │  保到最后          │  最难决策区（找替代方案）
      │  Color Grading     │  实时阴影、角色特效
      │  Bloom（适度）     │  抗锯齿
      │  角色贴图分辨率    │
  ─── ┼────────────────────┼────────────────────
  低  │  随时可关          │  第一个砍
      │  SSAO              │  级联阴影层数
      │  DoF（多数场景）   │  MSAA（移动端）
      │  Lens Flare        │  高精度 Bloom 分辨率
      └──────────────────────────────────────────
              低                    高
                        GPU 代价
```

**第一个砍**（高代价 + 低感知）：这是低档配置的首选关闭项，能明显释放 GPU 预算，玩家几乎不会注意到。

**保到最后**（低代价 + 高感知）：这些效果关掉会让玩家立刻感觉"这个游戏变味了"，但本身代价极低，没有理由关。

**找替代方案**（高代价 + 高感知）：不能直接关，但原始实现太贵。需要用低代价的近似方案替代，而不是在"开"和"关"之间二选一。

**随时可关**（低代价 + 低感知）：关了没什么影响，但也省不了多少，优先级低。

---

## 逐效果分析

### 实时阴影（Directional Light Shadow）

**GPU 代价**：高。Shadow Map 绘制会在 CPU 侧增加 Draw Call，在 GPU 侧增加渲染 Pass 代价。2048 分辨率的阴影在低端 GPU 上能吃掉 15–25% 的帧预算。

**玩家感知**：高。阴影消失是画面里最容易被注意到的变化之一，玩家会明确感知到"游戏画面变差了"。

**决策**：不要直接关——用替代方案。

| 方案 | 代价 | 效果 |
|------|------|------|
| 实时阴影（原始）| 高 | 最好 |
| Blob Shadow（圆形投影贴片）| 极低 | 差，但有"角色站在地上"的感觉 |
| 预烘焙 Lightmap | 低（运行时）| 静态场景好，动态物体无阴影 |
| 简化 Shadow Map（降分辨率 + 减 Cascade）| 中 | 视觉接受度较高 |

**建议**：低档用 512 分辨率 + 1 个 Cascade + 缩短 Shadow Distance（30m）。极低档用 Blob Shadow 完全替代实时阴影。不要在低档直接关掉所有阴影，这是感知代价最高的操作之一。

---

### SSAO（屏幕空间环境光遮蔽）

**GPU 代价**：高。在半分辨率下跑的 SSAO 仍然是一个完整的屏幕空间 Pass，通常占帧预算的 5–15%。

**玩家感知**：中低。SSAO 的主要作用是让物体接触地面的地方显得更"压实"，以及角落处的细微阴影。在手机屏幕的尺寸下，大多数玩家不会注意到 SSAO 关掉了，除非画面风格本身非常依赖体积感。

**决策**：中档可以考虑关，低档直接关。这是 GPU 代价最高、感知最低的效果之一，是低档配置最优先的释放项。

**例外**：写实风格的 MMO / 开放世界，玩家对空间感要求高，SSAO 关掉后画面会显得"飘"，感知敏感度升高。这类游戏中档也应考虑保留。

---

### Bloom（泛光）

**GPU 代价**：中。标准 Bloom 是多次降采样 + 高斯模糊 Pass，通常占 3–8% 帧预算。

**玩家感知**：高。Bloom 是移动游戏里最常被玩家"注意到存在"的效果——不是因为它显眼，而是因为关掉之后发光的物体（武器、技能、UI）会突然变得生硬、"干"。

**决策**：不要关，降质量。

低档 Bloom 的正确做法是降低 Bloom 的计算分辨率和迭代次数，而不是关闭：
- 高档：全分辨率，6 次迭代
- 中档：半分辨率，4 次迭代
- 低档：1/4 分辨率，2 次迭代
- 极低档：关闭，或只保留 UI Bloom

1/4 分辨率的 Bloom 代价极低，但玩家基本感知不出和高档的差异（Bloom 本身就是模糊效果，低分辨率反而"更合理"）。

---

### Render Scale（渲染分辨率缩放）

**GPU 代价**：高。Render Scale 从 1.0 降到 0.75，像素数量减少约 44%，GPU 填充代价线性下降。这是单个配置项里释放 GPU 预算最有效的手段。

**玩家感知**：低（移动端）。手机屏幕尺寸小、持握距离近，分辨率感知敏感度远低于 PC。从 1.0 降到 0.75，多数玩家在 6 寸屏幕上分辨不出来。

**决策**：优先砍。这是移动端低档配置里性价比最高的单个操作。

| 档位 | 建议 Render Scale |
|------|-----------------|
| 高档 | 1.0 |
| 中档 | 0.85 |
| 低档 | 0.75 |
| 极低档 | 0.65 |

**注意**：Render Scale 降低会让 UI 相对"更清晰"（UI 通常在原始分辨率渲染），但 3D 场景会显得"软"。在 UI 密集型游戏（卡牌、回合 RPG）里，玩家不容易注意到 3D 场景分辨率的变化。

---

### Color Grading / LUT

**GPU 代价**：极低。一次 LUT 采样 Pass，代价几乎可忽略不计（< 0.5ms）。

**玩家感知**：高。Color Grading 决定了游戏的整体色调和氛围。关掉之后，游戏会看起来"灰蒙蒙的"或"颜色怪怪的"，是破坏视觉风格最快的操作之一。

**决策**：全档保留，永远不关。

低档的正确做法是用**预烘焙的 LUT 贴图**替代运行时 Color Grading 计算，而不是关掉：
- 高档：完整的 Color Grading Stack（Lift/Gamma/Gain + Tonemapper + Vignette）
- 中档：保留，或简化为只跑 LUT
- 低档 / 极低档：预烘焙一张 LUT 贴图，运行时只做一次纹理采样

预烘焙 LUT 的代价和"什么都不做"几乎一样，但视觉风格完整保留。这是所有后处理里性价比最高的保留操作。

---

### 抗锯齿（Anti-Aliasing）

**GPU 代价**：中。TAA（Temporal AA）最重，SMAA 次之，FXAA 最轻，MSAA 在移动端 Tile-Based GPU 上代价低于 PC。

**玩家感知**：高（锯齿明显时）。锯齿在移动端屏幕上因为 DPI 高，比 PC 轻；但在细线条、长直边缘（建筑、武器边缘）上仍然明显。

**决策**：降方案，不直接关。

| 档位 | 建议 AA 方案 |
|------|------------|
| 高档 | TAA 或 SMAA |
| 中档 | FXAA 或 SMAA |
| 低档 | FXAA（代价极低）|
| 极低档 | 关闭（依赖 Render Scale 的自然模糊做软化）|

极低档可以关 AA，因为 Render Scale 0.65 的上采样本身会产生轻微模糊，缓解最明显的锯齿问题。

---

### 粒子特效密度

**GPU 代价**：高（动态）。高密度粒子系统对 GPU 的代价是动态的，很难精确预估——一场 Boss 战可能让帧率直接掉半。

**玩家感知**：高（ARPG / 动作游戏）/ 低（卡牌 / 休闲）。这是最受游戏类型影响的效果。

**决策**：按游戏类型差异化对待。

- **动作游戏**：粒子密度砍到 40% 时，玩家明显感知到技能"变弱了"。建议用 Flipbook 帧动画替代部分粒子，在低档保住核心技能的视觉冲击，砍掉背景装饰粒子。
- **卡牌 / 休闲**：粒子密度砍到 20% 玩家基本无感，可以激进压缩。

---

### LOD 切换距离

**GPU 代价**：高（LOD 保持高精度时）/ 低（积极切换时）。

**玩家感知**：低（远处）/ 高（近处）。

**决策**：远处 LOD 可以激进切换，近处 LOD 必须保护。

LOD Bias 从 1.0 降到 0.6，远处物体更早切到低 LOD，但近处 1–2 个 LOD 层级应该维持不变。低档配置里因为 LOD 切换过激导致"近处的树突然变成低模"，是玩家投诉质量下降的常见来源。

---

## 综合排序：从先砍到后砍

| 优先级 | 效果 | 操作 | 理由 |
|--------|------|------|------|
| 1（最先砍）| SSAO | 低档直接关 | 高代价，低感知 |
| 2 | Render Scale | 从 1.0 降到 0.75 | 高代价，移动端低感知 |
| 3 | Shadow Cascade 层数 | 从 4 降到 1–2 | 代价高，单层阴影视觉上可接受 |
| 4 | Shadow Distance | 从 80m 缩到 30m | 释放远处阴影代价 |
| 5 | 粒子密度 | 降至 40% | 按游戏类型调整激进程度 |
| 6 | Bloom 分辨率 | 降至 1/4，不关 | 低分辨率 Bloom 感知差异小 |
| 7 | LOD Bias | 从 1.0 降到 0.6 | 保护近处，放宽远处 |
| 8 | 抗锯齿方案 | 从 TAA 换 FXAA | 不关，只换低代价方案 |
| 9 | 实时阴影 | 换 Blob Shadow（极低档）| 不关，用替代方案 |
| 10（最后砍）| Color Grading | 换预烘焙 LUT，不关 | 代价极低，感知极高 |

**原则**：列表越靠后的效果，越不应该被关掉，而应该被"降质"或"换方案"。直接关闭是代价最高的操作，因为它通常比降质的感知代价大得多，但节省的 GPU 预算不见得更多。

## 替代方案思维

上面的分析反复出现"找替代方案"而不是"直接关"，这是值得单独强调的一种思维方式。

大多数高感知效果都有低代价近似版本：

| 效果 | 原始方案（贵）| 低档替代方案（便宜）| 感知差距 |
|------|-------------|-----------------|---------|
| 实时软阴影 | PCF / PCSS | Blob Shadow | 有感知，但可接受 |
| SSAO | 屏幕空间 AO | 烘焙 AO 贴图（静态）| 动态物体无 AO |
| 实时反射 | SSR | Reflection Probe（烘焙）| 静态场景几乎无感知 |
| 高质量 DOF | CoC 散景 | 简单径向模糊 | 明显但可接受 |
| Color Grading | 实时 Stack | 预烘焙 LUT | 几乎无感知 |
| 粒子特效 | 动态粒子系统 | Flipbook 帧动画 | 轻微感知 |

替代方案的价值在于：它允许你在"全开"和"全关"之间有一个中间档，让低档设备也能看到"这个效果存在"，只是精度低一些。

## 一个快速诊断方法

如果你正在为某个档位的配置纠结，试着用这个问题框架过一遍：

1. 这个效果关掉，用截图做 A/B 比较，能立刻看出差异吗？
2. 这个效果对这个游戏类型的核心体验有关联吗（ARPG 的技能特效 vs. 卡牌的背景场景）？
3. 这个效果有低代价的替代方案吗？
4. 关掉这个效果能释放多少帧预算（实际测量，不要靠猜）？

只有当"第 1 题答否"或"第 4 题答案是实质性提升"时，关掉才是合理的选择。其他情况，优先考虑降质而不是关闭。
