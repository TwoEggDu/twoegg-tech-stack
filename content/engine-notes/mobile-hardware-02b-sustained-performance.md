买手机时经常看到这样的评测：低端机刚开始跑分还看得过去，但玩 20 分钟游戏后就开始卡。这不是偶然，而是硬件设计的必然结果。旗舰和低端机的真正差距，不是第一分钟的峰值性能，而是第 30 分钟还剩多少性能。

---

## 峰值性能 vs 持续性能

移动端的性能指标有两个，大多数跑分软件只测第一个：

```
峰值性能（Burst Performance）：
  设备在温度正常、无热限制时的最大输出能力
  → 跑分软件通常只跑 1-3 分钟，测的就是这个

持续性能（Sustained Performance）：
  设备在连续高负载 15-30 分钟后能维持的性能
  → 这才是游戏体验的真实决定因素
```

**各档次的持续性能留存率（实测参考值，不同设备有差异）**：

| 档次 | 代表设备 | 峰值 GPU 性能 | 30 分钟后剩余 | 留存率 |
|------|---------|-------------|-------------|------|
| 旗舰（均热板） | iPhone 15 Pro | 100% | ~80-85% | **80-85%** |
| 旗舰（均热板） | 小米 14（骁龙 8 Gen 3） | 100% | ~70-75% | **70-75%** |
| 高端（热管） | 骁龙 8 Gen 1 机型 | 100% | ~55-65% | **55-65%** |
| 主流（石墨烯） | 骁龙 7 Gen 1 机型 | 100% | ~50-60% | **50-60%** |
| 低端（无散热结构） | 骁龙 6xx 机型 | 100% | ~35-45% | **35-45%** |

**结论**：低端机 30 分钟后可能只剩不到一半的 GPU 性能，帧率从 30fps 跌到 15fps 是常见情况。

## 散热硬件：决定持续性能的物理基础

手机没有风扇，散热完全依靠导热材料把热量从芯片传导到手机外壳，再由外壳散发到空气中。不同档次的散热硬件设计差异很大。

### 均热板（Vapor Chamber）

旗舰机的标配散热方案：

```
均热板工作原理：
  一个密封的扁平腔体，内部充有少量工作液体（通常是水）

  散热过程：
  SoC 发热 → 液体在热端蒸发（吸收大量热量）
           → 蒸汽快速扩散到整个腔体面积
           → 在冷端（远离 SoC 的区域）冷凝回液体
           → 液体通过毛细结构流回热端
           → 循环往复

优势：
  → 把 SoC 的点热源「摊平」到整个手机背板面积（约 40-60 cm²）
  → 导热效率约是铜片的 5-10 倍
  → 可以把 SoC 温度维持在较低水平，推迟降频触发时间
```

**面积是关键**：旗舰机的均热板面积通常在 3000-5000 mm²，高端游戏手机（如 ROG Phone）可以达到 6000 mm² 以上。面积越大，散热能力越强。

### 热管（Heat Pipe）

中高端机的常见方案，成本低于均热板：

```
热管 vs 均热板：
  热管：一维导热，只能沿管的方向传热
  均热板：二维导热，可以向整个面积扩散热量

  热管适合「把热量从 SoC 引到手机顶端或底端」
  均热板适合「把热量均匀扩散到整个背板」

热管 + 石墨烯的组合：
  热管把热量引走 → 石墨烯片负责在较大面积上扩散
  成本比纯均热板低，效果介于均热板和单独石墨烯之间
```

### 石墨烯贴片

主流机的常见方案：

```
石墨烯的特性：
  面内导热率：约 700-1500 W/(m·K)（铜约 400，铝约 200）
  厚度极薄（~0.1mm），适合堆叠在 PCB 上方

局限：
  只能在面内高速传导，垂直方向导热率很低（约 5-10 W/(m·K)）
  → 热量必须先从 SoC 传到石墨烯层，才能横向扩散
  → 与均热板相比，没有液冷的相变效应，散热峰值能力更弱
```

### 低端机：几乎没有主动散热结构

```
低端机的典型结构：
  SoC → 导热硅脂 → 金属中框

  没有均热板、没有热管、石墨烯面积很小
  → SoC 热量直接积累在芯片周围
  → 温度上升速度快 → 更快触发降频 → 持续性能极差
  → 表面温度容易超过 45°C（用户体验差）
```

## 内存发热与持续带宽

内存也是热源之一，而且内存的发热会影响它自身的持续工作带宽，这个细节经常被忽略。

### 高频内存的热问题

```
LPDDR5X 的发热情况：

LPDDR5X 运行在 8533 MHz，功耗约 0.5-0.8W
在手机内部，内存芯片紧贴主板，散热路径有限

温度对内存的影响：
  内存控制器在高温下会主动降低内存频率（类似 CPU 降频）
  LPDDR5X @ 正常温度：77 GB/s（满速）
  LPDDR5X @ 高温（>60°C 环境）：可能降至 LPDDR5 等效频率（51 GB/s）

旗舰机的应对：
  均热板的导热能力覆盖到内存区域，抑制内存发热
  整体热设计余量更大，内存很少触发降频

低端机（LPDDR4X）：
  虽然频率低（发热少），但散热结构也几乎没有
  LPDDR4X 本身带宽就低（17-34 GB/s），还有可能因为温度进一步降速
```

### 内存带宽与 GPU 性能的联动

在移动端，内存带宽是 GPU 和 CPU 共享的资源：

```
重度游戏场景下的带宽分配（骁龙 8 Gen 3，77 GB/s 总带宽）：

GPU（纹理读取 + RT 读写）：约 35-40 GB/s
CPU（场景逻辑 + 物理 + AI）：约 10-15 GB/s
NPU（如果有 AI 处理）：约 5 GB/s
系统 / 显示 / 其他：约 5-10 GB/s

总计：约 55-70 GB/s，接近上限

内存降频后（假设降至 51 GB/s）：
  GPU 可用带宽被压缩 → 纹理采样开始出现等待延迟
  → 表现：帧率下降，但 GPU 占用率看起来不高（实际是在等内存）
```

## 制程效率对持续性能的影响

同等算力下，制程越先进，功耗越低，发热越少，可以维持的性能就越久。

```
假设目标：持续输出 1 TFLOPS GPU 算力

12nm 制程（低端）：
  需要 ~4W GPU 功耗
  手机散热上限 ~1.5W（无散热结构）
  → 超出散热上限 → 很快触发降频 → 实际持续 GPU 性能只有 0.4-0.5 TFLOPS

4nm 制程（旗舰）：
  实现 2 TFLOPS 只需 ~5W GPU 功耗
  旗舰散热上限 ~6-8W（均热板）
  → 在散热允许范围内，可以长时间维持高性能
  → 30 分钟后帧率依然接近峰值
```

**这就是为什么旗舰机「跑久了还是快」**：不是因为它可以无限散热，而是制程先进 → 功耗低 → 散热压力小 → 触发降频晚 → 降频幅度小。

## 内存容量对持续体验的影响

除了速度，内存容量在长时间游戏过程中也会产生影响。

### 长时间游戏的内存累积问题

```
典型问题：游戏玩了 1 小时后开始变慢或闪退

原因分析：
  游戏运行过程中，部分资源未被正确卸载（内存泄漏或缓存策略激进）
  → 内存使用量逐渐增长
  → 低内存机型（可用内存 1-2 GB）：30-60 分钟后触发 OOM
  → 系统强制回收内存 → 游戏闪退

旗舰机（可用内存 8 GB）：
  → 同样的内存泄漏问题，但有更大的缓冲空间
  → 玩家可能玩几个小时都不会触发 OOM
  → 即使有轻微内存问题也不会立即崩溃
```

### 内存容量影响后台体验

```
场景：玩游戏时接微信消息，切回游戏

低内存机型（4 GB）：
  微信在后台被 OOM Killer 杀掉（内存不足）
  → 切回微信需要重新启动（冷启动，约 2-3 秒）
  → 切回游戏时，游戏可能也被重新加载（场景重新进入）

旗舰机（12 GB）：
  微信保持在内存中（热启动）
  → 切换几乎瞬间完成
  → 切回游戏时继续之前的状态，无需重新加载
```

## 对开发者的设计启示

**分级配置不能只看 GPU 算力**：

```
正确的分级判断依据：

旗舰档：
  ✓ 可以开高帧率模式（120fps）
  ✓ 可以开全分辨率 + 后处理
  ✓ 可以预加载更多场景资源（内存充裕）
  ✓ 30 分钟后帧率依然稳定，不需要主动降质

高端档：
  ✓ 60fps 稳定是合理目标
  ⚠ 建议监控热状态，在游戏 15-20 分钟后主动检查帧率
  ✓ 内存容量仍然充裕，可以保持正常资源预算

主流档：
  ⚠ 以 30fps 稳定为目标，60fps 只在轻场景开放
  ⚠ 控制单场景贴图总量（建议不超过 500MB）
  ⚠ 热稳定性差，建议接入 Thermal API 主动降质
  ⚠ 避免大量动态 AssetBundle 加载（存储速度慢）

低端档：
  ✓ 目标是"不闪退 + 30fps 基本稳定"
  ✓ 所有后处理关闭
  ✓ 贴图压缩到最低分辨率，控制内存占用在 800MB 以内
  ✓ 避免超过 1-2 个实时光源
  ✓ 长时间游戏需要考虑 OOM 防护（定期检查内存，主动 GC）
```

**持续性能是分级标准，不是峰值跑分**：

测试低端机时，不要只测开场 5 分钟的帧率。用以下流程测试：
1. 进入游戏，连续玩 30 分钟
2. 记录第 1、5、10、20、30 分钟的帧率曲线
3. 如果 30 分钟帧率 < 第 5 分钟的 60%，说明散热设计对该设备不友好，需要主动降质策略
