> 如果只用一句话概括这篇文章，我会这样说：很多资源不是“天生很重”，而是它们看起来像小开销，进项目以后却刚好踩中了最贵的几种组合方式。

这篇还是写给美术，不是写给渲染工程师。

它不解释底层图形管线，也不要求你先会看 Profiler。  
它只想建立一种更稳定的成本直觉：

`为什么有些资源单看没什么，放进真实项目后却突然变贵？`

## 真正危险的，通常不是“大”，而是“贵的方式不明显”

美术最容易被误导的，不是那些一眼就夸张的内容。  
真正容易出事的，往往是这些：

- 粒子不算多，但透明层很多。
- 不是全屏特效，但屏占比已经很大。
- 不是常驻重场景，但它刚好出现在最满的时候。
- 不是只有一个效果，而是它和阴影、后处理、角色发光、UI 一起出现。

所以很多资源的真实问题不是：

`它是不是特别大？`

而是：

`它是不是用了几种最容易叠成本的表达方式？`

## 透明为什么总是“看起来不重，却最容易出事”

透明最容易让画面立刻变漂亮，所以也最容易被团队当成安全默认项。

常见情况是：

- 柔边火焰
- 发光拖尾
- 半透明屏障
- 透明 UI 大底
- 外发光和边缘能量层

这些东西单独看，经常会让人产生一种错觉：

`也就这一层，应该没什么。`

问题在于，透明最怕的不是单层，而是：

- 叠层
- 大面积
- 长时间
- 和别的透明一起出现

所以透明最常见的危险信号不是“这个效果有透明”，而是：

`这个效果一旦开始铺屏、叠层、常驻，就很容易不再是小成本。`

## 阴影和后处理为什么最像“一个开关”，却最容易吃预算

很多美术会把阴影、Bloom、模糊、景深和屏幕空间效果理解成“画质增强项”。

这种理解的问题在于，它会天然把这些东西想成：

`多一点更好看，少一点就差一点。`

但在项目里，它们更像：

`非常容易在关键时刻整块吃预算的系统型效果。`

它们之所以危险，不是因为不能用，而是因为：

- 一旦打开，常常不是只增加一点点。
- 它们很喜欢和别的重内容一起出现。
- 它们经常被默认常驻，而不是按时机出现。

尤其当下面这些东西叠在一起时，风险会明显上升：

- 大面积透明特效
- 屏幕命中反馈
- 强 Bloom
- 模糊或色偏
- 角色本体外发光
- 场景本身的阴影和氛围效果

这也是为什么有些项目会出现一种很典型的体验：

`你说不清到底是哪一个东西把性能打穿了，但一到大招或演出时就是明显变差。`

因为那通常就不是一个东西的问题，而是一组会互相放大的表达，同时出现在了最糟的时刻。

## 屏占比为什么比“数量”更值得警惕

这是美术最需要建立的成本直觉之一。

很多人看到一个效果时，第一反应是：

`数量也不多。`

但项目里很多成本问题，关键根本不在“多少个”，而在：

`它覆盖了屏幕多大面积。`

例如：

- 一个很大的透明罩子
- 一片铺满画面的烟雾
- 一个接近全屏的受击闪白
- 一张大面积模糊背景
- 一个范围很广的技能爆发

这些东西哪怕“只有一个”，也可能比很多局部小效果更危险。

所以美术在判断时，最好把“数量”换成更接近真实体验的问题：

`它是不是会占掉玩家眼前大面积画面？`

## 为什么“单个没事，组合就炸”才是项目里的常态

项目里的多数性能事故，都不是因为某个资源一眼就坏掉了。  
而是因为很多“都还行”的东西，刚好在同一时刻一起出现。

最常见的几种组合是：

### 特效 + 后处理

技能爆发时，特效、Bloom、闪白、色偏、模糊一起出现。

### 角色发光 + 技能特效 + 场景氛围

角色本体已经带外发光，技能再叠一层，场景本身还有雾效和阴影。

### UI 叠层 + 模糊背景 + 动效

界面一开，页面本体、模糊大底、半透明遮罩和过渡动画一起拉起。

### 主城密度 + 植被 + 阴影

场景本身就已经很满，再叠上角色同屏、植被和完整阴影。

这些组合最容易把团队带进一个误判：

`这个也不算重，那个也不算重。`

问题是，玩家感受到的从来不是“单项解释”，而是这一帧、这一屏幕、这一时刻的总账。

## 为什么首次出现比持续出现更容易伤体验

很多资源之所以“看起来平时没问题”，是因为你后来看的已经不是第一次了。

真实玩家最容易遇到的危险时刻常常是：

- 第一次开这个页面
- 第一次放这个技能
- 第一次进这个场景
- 第一次见这个怪和这套特效

这类时候，资源的风险往往不只是美术表达本身，还会夹着：

- 首次准备
- 首次激活
- 首次上传
- 首次命中某些运行时链路

所以美术最该警惕的一种信号是：

`第二次还好，第一次明显不对。`

这通常说明问题并不只是“静态规格大”，而是：

`这个资源的第一次成本，被放在了一个很不该放的时机。`

## 美术最常见的四种成本误判

### 误判一：文件不大，所以应该不贵

不稳。  
文件大小只是其中一层，不代表真实运行时成本。

### 误判二：粒子不多，所以应该不重

不稳。  
透明、屏占比和叠层关系，经常更关键。

### 误判三：只是一层 Bloom / 模糊 / 发光，不会有什么问题

不稳。  
一层单开不一定危险，但它几乎从来不只单开。

### 误判四：第二次没事，所以说明资源没问题

也不稳。  
真实体验最容易被破坏的，恰恰就是第一次。

## 一张更实用的成本直觉表

| 表达方式 | 为什么容易被低估 | 真正该警惕什么 |
|------|----------------|--------------|
| 透明层 | 单层看起来不夸张 | 叠层、大面积、常驻 |
| 阴影 | 像一个普通画质开关 | 和场景密度、角色同屏一起出现 |
| 后处理 | 看起来只是气氛增强 | 和特效、受击、演出叠加时整块吃预算 |
| 大范围特效 | 数量不一定多 | 屏占比大、持续时间长 |
| 模糊背景 / UI 大底 | 看起来只是 UI 装饰 | 首开、切页、叠层时一起拉起 |
| 外发光 / 能量边 | 像角色气质的一部分 | 和技能特效、Bloom、场景氛围一起变满 |
| 首次出现的资源 | 第二次往往正常 | 第一次总在最敏感的时刻伤体验 |

## 结论

美术真正需要建立的，不是“绝对大小”的直觉，而是“隐藏成本”的直觉。

也就是：

- 哪些东西最容易靠透明和铺屏偷偷变贵。
- 哪些东西最容易像一个小开关，实际上整块吃预算。
- 哪些东西单开没事，一叠就会放大。
- 哪些东西平时没问题，但第一次最危险。

只要这几层直觉立起来，你看资源时就不会只剩一句：

`它看起来也没多大啊。`
