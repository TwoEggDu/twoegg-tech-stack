角色预算最容易被误写成一张表：

- 主角 3 万面
- NPC 1 万面
- 骨骼不超过多少

这当然比完全没有好。  
但它仍然不够。

因为角色从来不是只有面数这一笔账。  
它同时会吃：

- GPU 几何和材质账
- CPU 动画和蒙皮账
- 内存常驻账
- 包体和贴图账
- 同屏数量乘数

所以角色预算真正值钱的地方，不是“给一个数字”，而是建立：

`单体规格 x 同屏数量 x 目标档位 x 使用时机`

这套模型。

## 角色预算首先在管什么

一套真正能用的角色预算，至少要同时看下面这些对象：

- 面数
- 材质球数量
- 骨骼数量
- 蒙皮影响数
- 动画片段和状态机复杂度
- 主贴图、法线、遮罩和次级贴图
- 同屏数量

## 判断依据是什么

### 1. 角色通常是玩家最在意的内容对象

所以角色预算和场景预算不一样。  
很多场景细节可以降，角色主贴图、轮廓和关键识别信号通常更应该保。

### 2. 单个角色不重，不代表多角色同屏不重

角色预算最容易被低估的一点就是乘数效应：

- 一个角色过线，不代表十个角色还能过线
- 动画、蒙皮、阴影、材质分支都是按数量放大的

### 3. 角色很多时候同时吃 CPU 和 GPU

例如：

- 骨骼更多，CPU 或 GPU 蒙皮更重
- 材质更多，提交和绘制更碎
- 贴图更高，显存和包体更贵

所以角色预算本质上是典型的多桶预算对象。

## 一套更稳的角色拆法

### A. 几何规格

至少写清：

- 主角 / Boss / 精英 / 路人 NPC 各自的面数范围
- LOD 层级和切换距离

### B. 蒙皮规格

至少写清：

- 骨骼数
- 每顶点影响数
- 是否允许复杂次级骨骼

### C. 材质与贴图规格

至少写清：

- 材质球数量
- 主贴图尺寸
- 法线和遮罩是否保留
- 是否允许高价材质分支

### D. 动画与运行时规格

至少写清：

- 同时激活的动画层数
- 角色是否允许复杂 IK / 程序化修正
- 同屏最大角色数

## 最常见的误判

### 1. “角色预算就是面数预算”

不对。  
如果一个角色面数不高，但：

- 材质球很多
- 骨骼太多
- 法线和遮罩贴图很多
- 阴影投射和后处理参与太多

它仍然会很贵。

### 2. “主角过了预算，就把主角削掉”

很多时候更合理的做法是：

- 优先压非核心材质分支
- 压次级贴图
- 调整同屏数量
- 调整低档 LOD 和阴影策略

而不是先动最影响感知的核心识别信息。

### 3. “单体检测通过就行”

角色预算一定要看：

- 主角单独
- 小队同屏
- 群怪同屏
- Boss 战高特效叠加

因为玩家真正玩的不是“一个角色在空场景里”。

## 角色预算应该落成什么规则

我建议至少直接写出四张表：

- 主角 / Boss 规格表
- NPC / 小怪规格表
- 贴图与材质表
- 同屏数量与档位联动表

这样角色预算才会从“美术感觉差不多”变成工程规则。

## 结尾

所以如果只用一句话收束这篇，我会这样说：

`角色预算真正要管理的，不是一个面数数字，而是角色作为内容对象在几何、蒙皮、材质、动画和同屏乘数上的总成本；只有把这些账一起算，角色规格才不会在项目后期变成最难追回的一笔债。`
