场景预算真正难的地方，不是定一个数字，而是把“场景很重”翻译成一张能验收的表。

## 模板字段

| 项目 | 预算线 | 当前值 | 测试机位 / 路径 | owner | 超线动作 |
|-----|-------|-------|----------------|------|---------|
| 可见物数量 |  |  |  |  |  |
| Draw / 提交压力 |  |  |  |  |  |
| 动态灯光数量 |  |  |  |  |  |
| 阴影距离 / 阴影投射体 |  |  |  |  |  |
| Streaming 峰值 |  |  |  |  |  |
| 切场景双驻留峰值 |  |  |  |  |  |
| 大世界边界触发频率 |  |  |  |  |  |

## 为什么这些字段必须写

- 不写测试机位，场景预算很难复现
- 不写路径，streaming 和切换峰值就会漂
- 不写超线动作，最后只会变成“继续优化一下”

## 怎么判断这张表有用

- 团队能不能在同一个机位上复现问题
- TA、美术、客户端是否都能看懂
- 新场景接入时是不是能直接拿它做首轮验收

## 最短结论

场景预算模板的价值，是把“场景很重”翻译成：

`在固定机位、固定路径、固定设备上，哪些预算桶超了，谁来收。`
