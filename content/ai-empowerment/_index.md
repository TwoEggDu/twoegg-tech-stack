这一栏讲的不是 AI 能做什么，而是游戏开发团队**实际怎么用 AI 解决问题**。

每篇文章都来自真实项目的落地经验——20 人团队、Unity MMO、内网部署。不写 prompt 教程，不绑定特定产品，讲工作流设计和工程判断。

## 核心模型

整个系列围绕一个闭环展开：

```
┌───────────────────────────────────────┐
│  Dify — 团队唯一问答入口               │
│  策划/QA/开发都在这里问                 │
└──────────────┬────────────────────────┘
               │ 答不了 → 知识缺口
               v
┌───────────────────────────────────────┐
│  LKB + Claude Code — 知识生产引擎      │
│  读代码 + 读配置 + 读原始资料 → 文档    │
└──────────────┬────────────────────────┘
               │ 文档回流
               v
┌───────────────────────────────────────┐
│  Hugo Wiki — 知识沉淀层                │
│  Dify 索引 → 下次能答                  │
└───────────────────────────────────────┘
```

## 三条阅读线

- **知识管理线**（01-04）：从闭环总论到 Dify 问答层、LKB 知识生产、缺口回流机制
- **AI 上下文线**（05-06）：CLAUDE.md 项目上下文和 Skill 领域知识注入
- **开发工作流线**（07）：AI 辅助从协议到 UI 的跨层联动
