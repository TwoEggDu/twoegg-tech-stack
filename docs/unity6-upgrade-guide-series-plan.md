# Unity 2022 → Unity 6 升级决策指南系列规划

## 专栏定位

这组文章不是 Unity 官方 Upgrade Guide 的翻译，也不是逐条 API 变更清单。

它真正要解决的问题是：

`帮技术负责人做一个有依据的升级决策——升不升、什么时候升、怎么升、升完怎么验——每一步都有可量化的判断框架，而不是凭感觉。`

一句话说，这个专栏的重点不是"Unity 6 有什么新功能"，而是：

`升级这件事本身的成本收益分析和执行方法论。`

---

## 目标读者

- 正在做"升不升级"决策的技术总监或客户端主程
- 已经决定升级，需要一份系统性 checklist 的项目负责人
- 负责第三方依赖审计（HybridCLR、YooAsset、UniTask 等）的工程师
- 需要说服团队/管理层升级或不升级的技术决策者

## 系列边界

### 属于这个系列的内容

- 升级 vs 不升级的成本收益分析框架（包体、性能、人力、插件兼容、维护窗口）
- 按模块的兼容性排查流程（渲染、UI、物理、网络、音频、编辑器扩展）
- 版本锁定策略（6.0 LTS vs 6.x 滚动 vs 2022 LTS）与项目类型的匹配
- 核心第三方库的 Unity 6 适配状态审计

### 不属于这个系列的内容

- GPU Resident Drawer 等技术细节 → 在"渲染管线升级实战"系列
- Awaitable API 等技术细节 → 在"运行时与工具链变化"系列
- 完整的 URP 教程 → 已有"URP 深度"系列
- 完整的 HybridCLR 适配教程 → 已有 HybridCLR 系列
