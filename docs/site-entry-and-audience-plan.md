# 站点入口与人群分层规划

## 目标

这份文档只记录站点入口策略、人群分层和后续写作约束，不承担站点安装说明、仓库说明或总选题母表的职责。

这个站点按长期经营来设计，不按未来三个月的短期转化来排结构。公开层要稳定、克制、可持续；定向层和私密层则服务于不同关系阶段的沟通。

## 当前确认的决策

1. 公开站点只开放 `顾问合作` 这一条主入口。
2. `加入团队`、`合伙创业`、`投资人` 不作为公开首页并列入口出现。
3. `HR / 猎头` 不单独做公开一级入口；如有需要，做定向发送的快速筛选页。
4. 站点公开叙事不再同时对所有对象喊话，而是先把“我能进入复杂工程现场并做成可交付结果”这件事讲清。
5. 后续文章选题要服务于这套入口策略，先沉淀公开证据库，再按对象重组材料。

## 分层结构

| 层级 | 主要对象 | 是否公开 | 作用 |
|------|----------|----------|------|
| 公开层 | 顾问合作 | 是 | 建立统一外部认知，承接公开来访 |
| 定向层 | 加入团队、HR、猎头 | 否 | 通过单独链接发送，帮助对方快速判断是否继续推进 |
| 私密层 | 合伙创业、投资人 | 否 | 只在明确关系和语境下发送，避免在公开站点提前暴露 |

## 公开层的主叙事

公开站点的核心不是“我面向很多人”，而是：

> 我擅长进入复杂工程现场，快速判断问题、澄清边界、搭结构，并把复杂问题推进成可交付结果。

因此公开层要优先证明的是：

- 我能诊断复杂问题，而不是只会列知识点。
- 我能做工程判断，而不是只会写局部实现。
- 我能把一次性救火沉淀成系统能力，而不是只讲单次修复。
- 我做过真实交付，而不是只写抽象方法论。

## 定向层与私密层的定位

### 加入团队

不在公开首页直接暴露，但可以通过定向页回答下面这些问题：

- 你适合承担什么类型的职责。
- 你是否具备 owner 级别的判断和推进能力。
- 你做过哪些能代表长期工程建设能力的事情。
- 推荐先看哪几篇文章，能最快判断值不值得继续聊。

### HR / 猎头

`HR` 和 `猎头` 都属于筛选与转发角色，不需要拆成两个独立入口。更适合做成一个短、快、标准化的定向页，重点回答：

- 你是谁，核心标签是什么。
- 你适合什么岗位或合作类型。
- 最值得转发给技术负责人或用人经理的 3 篇内容是什么。
- 对方继续推进前，应该优先看哪一页。

### 合伙创业

这类材料不适合公开平铺。它关注的是长期判断、承担性、方向感、组织能力和共同做事的匹配度，不是普通作品集入口。

### 投资人

这类材料也不建议作为公开首页入口。投资人更关心长期判断、能力壁垒、执行模型和可验证的结构化认知，适合在明确语境下定向发送。

## 技术与暴露边界原则

如果后面要做“只有我发链接给对方才看得到”的页面，必须先区分两种情况：

- `不主动展示`
  这只是定向发送页，可以不放导航、不放首页、不进 sitemap，并加 `noindex`。
- `技术上受保护`
  这才是私密材料，需要访问控制，不能只靠“隐藏链接”。

如果站点和仓库本身是公开的，那么“隐藏页面”只能算未公开，不算真正私密。涉及合伙、投资、合作条件、阶段性判断的内容，不能把“没挂首页”当成保密措施。

## 公开文章的证据模块

公开文章优先积累下面四类证据：

1. `复杂问题诊断`
   证明你能从现象进入结构，知道该先怀疑哪一层、如何缩小范围、怎样判断根因。
2. `工程取舍与方案判断`
   证明你能比较方案、说明代价、讲清适用边界，而不是只给结论。
3. `平台化与长期治理`
   证明你能把工具链、质量门禁、资源链路、渲染/性能治理变成长期能力。
4. `真实交付案例`
   证明你做过什么、在什么约束下做、为什么这么做、最后带来了什么结果。

## 后续选题约束

未来文章不需要表面上分成“顾问版 / 入职版 / 合伙版 / 投资人版”四套栏目，而是围绕同一套公开证据库来写。

每个选题在进入排期前，至少要回答下面一个问题：

- 这篇能不能证明我的诊断能力？
- 这篇能不能证明我的工程判断？
- 这篇能不能证明我的系统建设能力？
- 这篇能不能证明我的真实交付能力？

如果一篇文章同时不支撑以上四类能力，也很难被后续定向材料复用，那它的优先级就应该降低。

## 后续材料的重组原则

基于同一批公开文章，后面可以重组出不同的定向材料：

- `顾问合作`：强调问题诊断、边界澄清、落地推进。
- `加入团队`：强调 owner 能力、跨团队协作、长期建设。
- `HR / 猎头`：强调快速筛选、角色标签、推荐阅读路径。
- `合伙创业`：强调长期判断、承担性、方向感和组织能力。
- `投资人`：强调结构化认知、能力壁垒和执行模型。

也就是说，未来不是继续扩公开入口，而是先把可复用的公开证据写扎实。

## 当前阶段的执行原则

1. 公开站点继续围绕顾问合作主叙事收敛。
2. 公开文章优先补“诊断、判断、系统、交付”四类证据。
3. `加入团队`、`HR / 猎头`、`合伙创业`、`投资人` 的材料先按规划保留，不急着公开挂出。
4. 真正敏感的材料以后单独决定是否做访问控制，不默认放进公开站点。
