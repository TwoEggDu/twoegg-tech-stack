## 2026-04-12 — 技术站分类与定位决策

**问题：** Engine Notes 栏目承载 688 篇文章（全站 94%），读者无法快速定位内容，导航结构无法体现能力维度。

**约束：**
- 站点目标是隐性展示技术实力，面向百万年薪级岗位
- 已有 50+ 系列专题，物理拆分涉及 URL 变更
- Problem Solving（7 篇）和 Essays（4 篇）内容有价值但体量不足

**决策：**
1. 按能力维度拆分 Engine Notes 为：渲染管线 / 引擎工具链 / 性能工程 / 系统设计
2. Problem Solving + Essays 保留为独立栏目，不降级 — 技术决策类内容是高级岗位的核心差异化
3. 首页语气从 "自我证明" 改为 "内容导向"，让深度自己说话
4. 导航改为中文，与内容语言一致

**为什么：** 百万级岗位看的不是 "会什么"，而是 "怎么判断"。导航结构本身就是能力地图，每个栏目对应一个面试官关心的维度。

**技术域：** 站点架构

## 2026-04-12 — Header 布局调整

**问题：** 导航从 6 项增加到 9 项后，一行排不下，"工程判断"和"关于"掉到第二行。

**约束：**
- 9 个导航项必须全部可见（不折叠）
- 搜索栏需要保留
- 不能牺牲可读性

**决策：** 采用两行 header 布局 — 第一行放品牌标题 + 搜索栏，第二行放导航栏独占全宽。同时去掉品牌 tagline 让第一行更紧凑。

**为什么：** 搜索栏和 9 个导航放同一行空间不够，拆成两行后导航有充足空间一行排下，搜索栏和品牌在同一行也不会有大面积空白。

**技术域：** 前端布局

## 2026-04-15 — 站点故事主线确认

**问题：** 10 个栏目平铺，读者无法快速识别站主的核心能力。

**决策：** 确认故事主线为 **交付工程 + 长线运营**，这两个方向是核心差异化。渲染管线、引擎工具链、性能工程、系统设计是技术底座，代码质量、问题拆解、工程判断是补充证明。

**导航优先级：** 交付工程 → 长线运营 → 渲染管线 → 引擎工具链 → 性能工程 → 系统设计（6 项核心导航），其余栏目通过首页卡片进入。

**为什么：** 百万级岗位看的是"你能交付什么"，不是"你会什么技术"。交付工程和长线运营直接回答这个问题，技术深度是支撑证据。

**技术域：** 站点定位
