## 最值得优先补的 10 个数字

- 打包耗时
- 补丁包体积
- 发布资产体积 / 下载体积
- SDK 接入周期 / 初始化成功率
- Android / iOS 出包耗时 / 失败率
- 自动化性能测试覆盖场景数 / 回归提前发现次数
- 新技术从 0 到首版交付周期
- 特效检查覆盖资源量 / 问题拦截数
- 导表或资源处理的人时
- 发布失败率
- 新人上手时间
- 跨部门问题闭环时效
- 协作事故或漏提问题次数

## 数字不完整时怎么写

如果你拿不到精确数字，至少写成区间：

- 从“半天级”缩短到“小时级”
- 从“需要多人协作”缩短到“单人可执行”
- 从“高频人工检查”变成“工具自动校验”
- 从“上线前手动拼装”变成“标准化产物输出”
