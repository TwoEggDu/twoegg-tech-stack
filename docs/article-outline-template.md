# TwoEgg 文章大纲模板

> 用法：先判断文章类型，再套对应模板。默认优先用 `原理篇模板`。

## 1. 原理篇模板

适用：

- 架构篇
- 原理篇
- 系列主线文章
- 大多数“先抽象、后落地”的技术文章

```md
# 标题

> 如果这篇只记一句话，我建议记这个：`一句最短判断`

## 这篇在解决什么问题

- 这篇想回答什么
- 这个问题为什么会在项目里反复出现
- 常见但粗糙的理解是什么

## 为什么这个问题经常会被说粗 / 写歪

- 项目里的典型误解
- 为什么表面现象不是问题本体

## 先把问题拆成几层

- 层 1：
- 层 2：
- 层 3：

## 抽象模型

- 最小对象 / 最小链路 / 最小状态机
- 每个部分分别负责什么
- 哪些概念最容易被混掉

## 一个更稳的最小实现骨架

- 用伪代码、结构图或最小数据模型解释

## 放到具体引擎里怎么落

### Unity

- 在 Unity 里对应什么
- 常见实现方式
- 最容易踩的坑

### Unreal / GAS / 自研

- 对应什么思想或组件
- 和 Unity 的关键差异

## 一个坏实现通常怎么坏

- 坏法 1：
- 坏法 2：
- 坏法 3：

## 工程边界与取舍

- 哪些能力应该独立成层
- 哪些能力不应该被这个系统吞掉
- 多人 / 工具链 / 回归测试 / 性能会在哪些点改变设计

## 这篇真正想留下来的结论

`一句最短结论`
```

## 2. 映射篇模板

适用：

- Unity vs Unreal
- 自研方案 vs 框架方案
- 某类系统在不同引擎里的思想映射

```md
# 标题

> 这篇最短的一句话：`一句总判断`

## 先说清这是在比较什么问题

- 共同问题空间是什么
- 为什么不能直接拿 API 一一对照

## 先给共同抽象模型

- 不依赖具体引擎，先定义对象、边界、链路

## Unity 里通常怎么落

- 关键对象
- 关键链路
- 典型优点和代价

## Unreal / GAS / 自研里对应什么

- 对应对象
- 对应链路
- 真正的思想映射

## 它们真正的差别在哪

- 差在抽象边界
- 差在内建能力
- 差在工程约束

## 项目里怎么选

- 什么时候更适合 A
- 什么时候更适合 B
- 哪些前提不满足时不要硬上

## 最短结论

`一句最短结论`
```

## 3. 案例篇模板

适用：

- 排障
- 事故复盘
- 故障定位

```md
# 标题

> 这篇如果只记一句话：`一句总判断`

## 现象

- 用户/项目里看到了什么
- 为什么这个现象很容易误导

## 先别急着修：这类问题一般站在哪几层

- 层 1：
- 层 2：
- 层 3：

## 这次是怎么一步步定位的

- 第一步排除了什么
- 第二步确认了什么
- 第三步为什么缩到了某一层

## 真正根因

- 根因到底是什么
- 为什么表面看起来像别的问题

## 修法

- 最小修复是什么
- 为什么这个修复有效

## 如何防止再来一次

- 监控 / 约束 / 校验 / 回归

## 最短结论

`一句最短结论`
```

## 4. 索引篇模板

适用：

- 系列索引
- 阅读地图

```md
# 标题

> 一句话说明这组内容到底在解决什么。

## 这篇要回答什么

- 已覆盖什么
- 先读哪篇
- 遇到问题跳哪篇
- 接下来还会写什么

## 先给一句总判断

- 用一句话概括这整个系列的主线

## 推荐阅读顺序

0. 索引本身
1. ...
2. ...

## 按主题分组去读

## 如果你不是系统读，而是带着问题来查

### 问题 1

- 对应文章 A
- 对应文章 B

## 这组文章接下来还没写到哪

## 最短结论

`一句最短结论`
```

## 5. 快速检查

开写前先问自己：

- 这是原理篇、映射篇、案例篇还是索引篇
- 这篇文章的问题空间是什么
- 这篇文章的抽象模型是什么
- 这篇文章最后要落到哪个具体实现
- 这篇文章最容易写歪成什么
