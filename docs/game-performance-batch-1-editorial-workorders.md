# 游戏性能判断系列 Batch 1 编辑工作单

## 编辑工作单

**选题：** 性能-美术-01｜美术资源性能自检表：贴图、材质、LOD、UI、特效提交前先看什么
**系列 / 子系列：** 系列三·E / 性能-美术-01

---

### 1. 系列职责
把性能风险前移到资源提交前，让美术先会做“自检和避坑”，再进入技术解释文。

---

### 2. 与相邻文章的重复风险

| 相邻文章 | 重叠风险点 | 本篇处理方式 |
|---------|-----------|------------|
| 前一篇：[游戏性能判断入口｜美术先看资源预算、自检指标和高风险效果](../content/engine-notes/game-performance-entry-artist.md) | 都会讲“美术该先看什么” | 入口页只负责导航和分流，本篇给提交前可执行清单 |
| 后一篇：[每档资产规格清单：贴图压缩、LOD 与包体分层](../content/engine-notes/device-tier-asset-spec-texture-and-package.md) | 都会讲贴图、LOD、包体这些资源维度 | 本篇只讲“先查什么、什么现象危险、什么需要回头确认”，具体规格数字和分档表留给后文 |

---

### 3. 本文必须回答的核心问题
> 资源提交前先查哪几项，最能避免性能翻车？

---

### 4. 本文明确不展开的内容

- `CPU bound / GPU bound` 的判断流程和 Profiler 证据链
- `Draw Call / Overdraw / Pass / Culling` 的底层实现原理
- 各档位的精确规格表、压缩格式细节和平台 API 差异

---

### 5. 推荐二级标题结构

```
1. 为什么美术需要一张提交前自检表  — 职责：先把“这篇不是讲底层原理，而是讲交付前避坑”说清楚
2. 贴图和材质先看什么  — 职责：讲分辨率、透明、阴影、材质槽位、变体风险这些最常见入口
3. 模型、LOD 和场景资源先看什么  — 职责：讲轮廓、面数密度、LOD 切换、场景堆叠和远近关系
4. UI、特效和全屏效果先看什么  — 职责：讲首开、叠层、全屏覆盖、连续播放、透明叠层这些高风险区
5. 哪些东西“看起来不重”，其实最容易出事  — 职责：先给成本直觉，但不展开到底层渲染机制
6. 提交前 10 项自检表  — 职责：把全文收成一张可直接复用的清单
```

> 4~6 个，每节职责不重叠，顺序体现“先回答交付问题，再接到后续技术文章”的逻辑。

---

### 6. 读者前置知识

| 必须掌握 | 了解即可 |
|---------|---------|
| 贴图、材质、模型、UI、特效这些资源类型 | `LOD`、透明、后处理这些词的大致含义 |
| 知道项目存在高 / 中 / 低档设备差异 | 不同机型预算为什么会不同 |

### 8. 写法约束

- 开头不要从术语讲，直接从“资源做完了，不等于它能安全进项目”切入
- 每节都按“先看什么 -> 危险信号 -> 常见误判 -> 该找谁确认”展开
- 少讲“GPU 为什么这样算”，多讲“什么现象说明这份资源危险”
- 结尾必须落一张可执行的检查表，否则这篇就只是在讲道理
