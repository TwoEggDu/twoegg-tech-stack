## 扩展架构

### 新增板块结构

在现有 8 个板块基础上，新增/重组为以下结构：

```
现有板块（保留）：
├── 渲染管线（rendering）
├── 引擎工具链（engine-toolchain）
├── 性能工程（performance）
├── 系统设计（system-design）
├── 代码质量（code-quality）
├── 问题拆解（problem-solving）
├── 工程判断（essays）
└── 项目交付（projects）

需要强化的方向（在现有板块内扩展 + 新建系列）：
├── 后端深化 → system-design 内新系列
├── 策划/游戏设计 → system-design 内新系列
├── 美术管线 → 新板块或 rendering 子入口
├── 运营 → system-design 内新系列
├── 运维/DevOps → engine-toolchain 或新板块
├── 质量把控 → code-quality 强化 + 新系列
├── 团队管理 → essays 强化 + 新系列
└── 项目案例 → projects 强化
```
