## 写作约束

- 保持长文解释风格，但要持续服务工程判断。
- 不把“计算机基础”单独写成教材，而是嵌进游戏运行时问题里解释。
- 优先回答：为什么会慢、慢在哪、什么事不能在什么时候做、该先怀疑哪一层。
- Unity / Unreal 只做原理映射，不先掉进 API 细节。
