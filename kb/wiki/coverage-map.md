# Coverage Map — 概念在已发布文章里的覆盖地图

> 这是本 KB 的核心特色文件。回答两类问题：
> 1. 「概念 X 我之前在哪几篇文章讲过、各篇侧重什么角度？」
> 2. 「我要写新一篇 Y，已被前文覆盖的角度有哪些？哪些角度还没人讲过？」
>
> 维护节奏：每次发布新文章后 `publish-sync`，由 AI 自动追加。

## 使用方式

- 写新文章前 → 查目标概念在 map 里的现有覆盖 → 决定本篇侧重角度
- 重构系列时 → 看 map 上的孤立条目 → 决定要不要补 / 删 / 合并
- col-next / col-publish-audit 后续可读本文件辅助决策

## 格式约定

每个一级标题是一个**概念**或**实体**：

```markdown
## <概念名>

- [文章标题](content 相对路径) — 侧重 <角度>
- [文章标题](content 相对路径) — 侧重 <角度>
- 待补角度：<列出未覆盖的角度>
```

排序：按概念字典序。
